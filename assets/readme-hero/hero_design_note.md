# README Hero Design Note

## Project summary

This Codex skill lets agents inspect eligible pre-2022 GitHub repositories as untrusted, read-only design material without copying or executing code.

## Target users

Codex agents that need design inspiration from old repositories while preserving security and originality boundaries.

## Core problem

External repositories can introduce prompt-injection, supply-chain, copying, execution, and external-link risks when treated too casually as inspiration sources.

## Core workflow

Search old repositories, filter by age and eligibility, inspect static files only, then re-express useful ideas independently.

## Hero tagline

A safety-bounded Codex skill for turning old GitHub repositories into read-only design inspiration.

## Visual concept

A bright archival safety funnel where old repository folios move through age filtering, quarantine, locked safety barriers, risk diversion, and idea-only extraction.

## Material language

Ivory archival paper, copper rails and locks, matte enamel safety gates, red wax-like quarantine seals, and amber resin idea crystals.

## Image keywords

premium workflow visual, legacy repositories, read-only archive, safety gates, quarantine seals, idea extraction

## Generation method

Direct project-specific text-to-image generation using a premium art-directed product-object workflow direction. The prompt used archival paper, copper, enamel, wax-seal, and resin materials so the image reads as safe old-repo inspection rather than a generic tech diagram.
